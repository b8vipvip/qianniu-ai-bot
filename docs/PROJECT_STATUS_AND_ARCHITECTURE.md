# 项目进度、总架构与功能清单

最后更新：2026-07-20

本文档是当前生产架构、能力边界和开发进度的统一入口。千牛 UIA / IMSDK 研究细节继续参见 `docs/QIANNIU_CHAT_AUTOMATION_PROGRESS.md` 和 `docs/QIANNIU_MESSAGE_LIFECYCLE_VALIDATION.md`。

## 当前阶段

项目已形成“Windows 千牛 Bot + Ubuntu API 控制面 + 企业微信双向人工接管”的完整主链路：

- Windows Bot 接收千牛消息、聚合连续短消息、调用 AI 并发送回复；
- Ubuntu 控制面管理 AI 供应商、模型、测试、Bot 客户端和人工回复队列；
- 企业微信接收转人工通知，并通过加密回调把人工回复送回正确的 Windows Bot；
- 企业微信敏感参数在网页配置，经 Fernet 加密后保存到 SQLite，不从 `.env` 读取；
- Ubuntu 新容器、企业微信测试消息和加密回调已完成实机验收；
- API control plane CI、Windows CI、Windows x64 release build 均通过。

UIA v2 消息生命周期仍属于旁路研究能力，尚未替换生产接收主链路。

## 总体架构

```text
千牛聊天页面
  → 注入脚本 / 本地 WebSocket / CDP-QN 事件
  → 当前卖家与买家识别
  → 连续消息聚合、去重、草稿失效
  → 本地知识库或 AI 供应商
  → 重复答案与发送前稳定检查
  → QNRpa / FlaUI / UI Automation
  → 确认目标会话并发送给买家

Windows WeComAppBridgeClient
  ↔ HTTPS 人工回复任务队列
Ubuntu FastAPI 控制面
  ↔ 企业微信应用消息与加密回调
企业微信自建应用 / 个人微信插件入口
```

## Windows Bot

### 消息接收

当前生产接收链路以注入脚本、本地 WebSocket 和 CDP/QN 事件为主。

关键代码：

- `src/Bot/ChromeNs/MyWebSocketServer.cs`
- `src/Bot/ChromeNs/CDPClient.cs`
- `src/Bot/ChromeNs/QN.cs`
- `src/Bot/ChromeNs/MyOpenAI.cs`

### 连续消息协调

`BuyerMessageBurstCoordinator` 按“客服账号 + 买家”隔离消息轮次：

- 聚合一句话拆多条、补充、纠错和连续催促；
- 相邻重复文本去重；
- 后一条完整文本可替代前一条短前缀；
- 同一轮只生成一条回复；
- AI 生成期间买家补发时，旧草稿立即失效；
- AI 返回后执行发送前稳定检查；
- 取消、过时或发送失败的答案不会进入知识学习。

### AI 与知识库

- OpenAI 兼容 API；
- 多供应商、多模型和失败切换；
- 文本与视觉模型；
- 本地知识优先匹配；
- 知识搜索、分类、编辑、启停和删除；
- 文本、HTML、图片资料的 AI 智能导入；
- JSON 追加或覆盖导入导出；
- 真实发送成功后才触发知识学习。

### 消息发送

生产发送链路使用 `QNRpa`、FlaUI 和 Windows UI Automation：

```text
确认卖家与买家
  → 切换正确会话
  → 定位输入框并写入
  → Enter 或发送按钮
  → 输入框清空或消息回显确认结果
```

关键代码：

- `src/Bot/ChromeNs/QNRpa.cs`
- `src/Bot/ChromeNs/QN.cs`

## Ubuntu API 控制面

控制面位于 `services/api-control-plane/`，使用 FastAPI、静态管理页面、SQLite、Docker 和 Docker Compose。

主要能力：

- AI 供应商、Base URL、协议、模型和密钥管理；
- 文本模型与视觉模型探测；
- 普通、深度和计划任务测试；
- 实际路由、协议、耗时和结果记录；
- Windows Bot 客户端令牌管理；
- 人工回复任务绑定、短租约领取和完成回报；
- 企业微信网页配置和应用测试消息；
- 企业微信工单、通知、回调和回复队列。

生产数据库：

```text
/data/api-control-plane.db
```

升级时必须保留 `.env` 中的基础运行参数与加密主密钥，以及整个 `data/` 目录。

## 企业微信双向人工接管

### 网页配置

控制面“企业微信”页面支持：

- CorpID、应用 Secret、AgentId；
- 接收成员和允许回复成员 UserID；
- 工单有效期；
- 回调 Token 与 43 位 EncodingAESKey；
- 固定回调 URL；
- 企业微信应用测试消息。

安全设计：

- Secret、Token、EncodingAESKey 使用 Fernet 密文保存；
- 管理接口不回显敏感明文；
- 空白敏感字段表示保留原值；
- 保存后立即加载，无需重启；
- `API_KEY_ENCRYPTION_KEY` 必须长期固定并纳入备份。

### 转人工与回复

```text
Windows Bot 创建 QN-XXXXXXXX 工单
  → 控制面发送企业微信应用消息
  → 人工按“QN-XXXXXXXX 回复内容”回复
  → 签名校验和 AES 解密
  → 成员、类型、有效期检查
  → 写入绑定客户端的回复队列
  → Windows Bot 领取任务
  → 确认正确卖家/买家并发送
  → 发送成功后完成任务并学习知识
```

通知中的文本按顺序保留；图片、视频、语音、表情、文件和位置只使用安全占位符；手机号和疑似 API Key 在服务端脱敏。

### P1：UIA v2 Shadow Mode

- 先只做旁路观察，不触发 AI 或发送；
- 与现有 CDP/WebSocket 消息按稳定键交叉去重；
- 收集足够证据后再决定是否作为备用接收源。

### P2：工程化

- FastAPI 启动事件迁移到 lifespan；
- 数据库备份恢复脚本；
- 加密主密钥轮换方案；
- 端到端自动化回归；

## 风险与边界

- 千牛升级可能改变 DOM、AutomationId、窗口结构或发送行为；
- 自动发送必须再次确认目标会话，避免串单；
- 退款、投诉、赔偿和隐私问题应优先人工处理；
- 加密主密钥丢失或变化会导致敏感配置无法解密；
- 研究工具不得直接接入生产自动发送；
- 所有升级必须先备份并保留用户数据。
